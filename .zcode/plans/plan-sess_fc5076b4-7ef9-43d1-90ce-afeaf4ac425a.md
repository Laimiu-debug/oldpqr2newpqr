# PQR 文档转化工具 — 实现计划

## 总体架构
Python + Tkinter 桌面程序，使用 **Word COM 自动化（pywin32）** 读写文档（已验证可打开加密 `.doc`），把旧版 PQR（7表格结构）数据提取后填入新模板（2表格结构），输出 `.docx`。

```
项目结构:
D:\PQR转化项目\
├── PQR新格式空白.docx          (模板，只读不修改)
├── PQR18-01*.doc ...           (待转化文档)
├── 输出\                        (生成的docx，自动创建)
├── logs\                        (转化报告，自动创建)
├── convert_core.py             (核心: 字段提取 + 模板填充)
├── field_map.py                (字段映射配置 - 可调)
├── gui.py                      (Tkinter可视化前端)
├── run.bat                     (双击启动)
└── README.md                   (使用说明)
```

## 核心设计

### 1. 字段提取（convert_core.py）— 混合模式
对旧文档每个表格建立 **关键字定位优先 + 坐标回退** 的提取器：

```python
def extract_field(table, keywords, fallback_rc, label):
    """1) 扫描所有单元格，找含关键字的单元格→取同行右侧/下方值
       2) 找不到→回退到 (行,列) 坐标
       3) 都失败→记入 issues"""
```

提取源（旧文档）→ 目标（新模板）的映射分三块：

**A. 封面信息**（来源：旧文档表格1 + 段落）
| 旧文档 | 新模板位置 |
|---|---|
| 报告编号 PQR18-01 | 表格1 R3 / 表格2 R61 |
| 指导书号 pWPS18-01 | 表格1 R4 / 表格2 R3,R61 |
| 焊接方法 SMAW | 表格1 R5 / 表格2 R5,R62,R165 |
| 材质/规格 Q345R/δ4 | 表格1 R6 |
| 评定日期 | 表格1 R7 |

**B. pWPS 区块**（来源：旧表格3、4）→ 模板表格2 R10-R56
- 母材类别号/组别号、材料代号/标准、厚度范围
- 填充金属类别/标准/型号/牌号/规格
- 预热温度、道间温度、热处理温度/时间
- 保护气体、焊接位置
- 电特性表（焊道/方法/电流/电压/速度/热输入）
- 技术措施

**C. PQR 区块**（来源：旧表格5、6、7）→ 模板表格2 R59-R160
- 母材、填充金属（实测）、电特性（实测值，如 ①130②150）
- 拉伸试验 2行数据 → R110-R111
- 弯曲试验 4行（2面弯+2背弯）→ R116-R119
- 冲击试验 6行（3焊缝+3热影响区）→ R124-R129
- 金相、无损检测、附加说明、结论
- 焊工姓名/代号/施焊日期 → 附加说明区

**D. WPS 区块**（来源：复用 pWPS 数据）→ 模板表格2 R161-R218
WPS 范围值直接从 pWPS 的实测值推导（或直接复用），与 pWPS 结构对称。

### 2. 模板填充
- 每次转化：复制 `PQR新格式空白.docx` → 临时副本 → 用 COM 打开 → 按 `(行,列)` 写入单元格 → 另存为新文件名 → 关闭
- 用 `cell.Range.Text = 值` 写入（保留原格式）
- 复选框处理：旧文档用 `■`/`☑` 标记，新模板用 `□`/`☑`，按旧文档勾选状态替换
- 复合单元格（如"焊接方法：  SMAW"）：用正则替换冒号后的值

├─────────────────────────────────────────────┤

│ │ ✓ PQR18-01 转化完成                      │  │

- 文件夹选择后自动列出所有 `.doc`/`.docx`（排除模板）
- 列表支持搜索框过滤、多选
- 命名格式变量：`{编号}`、`{焊接方法}`、`{原文件名}`
- 后台线程转化，进度条+实时日志不卡界面
- 异常标记 `⚠`，成功 `✓`，失败 `✗`

### 4. 异常处理与汇总报告
- 每份文档的未匹配字段记入 `issues` 列表
- 转化结束后在 `logs\转化报告_20260711_HHMMSS.csv` 输出：
  - 文件名 | 状态 | 未匹配字段列表 | 备注
- 界面"查看转化报告"按钮直接打开最新报告

### 5. 依赖
- pywin32（已装）— Word COM
- tkinter（内置）— GUI
- 无其他外部依赖，纯标准库

## 实现步骤（落地顺序）
1. 写 `field_map.py` — 集中管理"旧字段关键字→模板坐标"映射表，方便日后微调
2. 写 `convert_core.py` — `extract_data(doc)` 返回结构化 dict + issues；`fill_template(data, out_path)` 填充模板；`convert_one(in_path, out_path, naming)` 主入口
3. 写 `gui.py` — Tkinter 界面，后台线程调度批量转化
4. 用 PQR18-01 实测转化，核对输出
5. 写 `run.bat`（双击启动）和 `README.md`

## 验收标准
- 3份样本文档都能生成 `.docx`，封面、pWPS、PQR、WPS 四块关键字段正确填充
- 加密文档能正常打开转化（依赖本机 Word 已记住密码/可打开）
- 未匹配字段在日志和报告中可见
- 200+文档批量转化界面不卡顿
