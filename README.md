# PQR 文档批量转化工具

把旧版焊接工艺评定报告（PQR，7 表格结构的 `.doc`）批量转化成新格式模板（2 表格结构的 `.docx`）。

## 快速开始

1. **双击 `run.bat`** 启动程序（或命令行执行 `python gui.py`）。
2. 点击「选择文件夹」，定位到存放 200+ 份 PQR 文档的目录。
3. 在左侧列表勾选要转化的文档（可用搜索框过滤，或「全选」）。
4. 设置「命名格式」和「输出目录」（默认 `输出\`）。
5. 点击「▶ 开始转化」，实时查看日志和进度。
6. 转化完成后点「查看转化报告」查看未匹配字段，点「打开输出目录」取走生成的 `.docx`。

## 命名格式

支持以下变量（区分大小写）：

| 变量 | 含义 | 示例 |
|------|------|------|
| `{编号}` | 报告编号 | `PQR18-01` |
| `{焊接方法}` | 焊接方法 | `SMAW` |
| `{原文件名}` | 原文件名（不含扩展名） | `PQR18-01（δ4，SMAW…）` |

示例：`{编号}_{焊接方法}` → `PQR18-01_SMAW.docx`

> 注意：若报告编号本身已含 "PQR" 前缀，命名格式里不要再加 "PQR"，否则会变成 `PQRPQR18-01`。

## 程序文件说明

| 文件 | 作用 |
|------|------|
| `gui.py` | 可视化前端（Tkinter），双击 `run.bat` 即调用 |
| `convert_core.py` | 转化核心：字段提取 + 模板填充 |
| `field_map.py` | 字段映射配置（旧字段 → 新模板单元格），**可自行微调** |
| `PQR新格式空白.docx` | 新格式模板（只读，不会被修改） |
| `输出\` | 生成的 docx（自动创建） |
| `logs\` | 转化报告 CSV（自动创建） |

## 转化报告

每次转化后在 `logs\` 生成一份 CSV（Excel 可直接打开），记录每份文档：
- 文件名、状态（成功/部分成功/失败）
- 未匹配字段数及字段列表
- 失败原因

状态说明：
- **成功**：所有字段都匹配填入
- **部分成功**：有字段未匹配（已在文档中留空），需人工补填
- **失败**：打开或保存出错

## 关于加密文档

本工具通过本机已安装的 Microsoft Word（COM 自动化）打开文档。**加密的 `.doc` 只要在这台电脑上能用 Word 正常打开，本工具就能处理**（依赖系统已缓存密码或已记住凭据）。若文档弹出密码输入框，请手动输入一次让系统记住。

## 环境要求

- Windows + Microsoft Word（已安装并可启动，已测试 Office/WPS 均可由 COM 调用）
- Python 3.8+，已安装 `pywin32`：
  ```
  pip install pywin32
  ```
  （Tkinter 随 Python 自带，无需额外安装）

## 字段映射微调

若某份新文档结构略有差异导致字段取错，编辑 `field_map.py`：
- `COVER_FIELDS`：封面字段
- `PWPS_FIELDS`：pWPS 区块（母材/填充金属/预热/热处理/气体/电特性/技术措施）
- `PQR_FIELDS`：PQR 主体区块
- `MECH_TEST`：拉伸/弯曲/冲击试验数据行
- `METAL_TEST`：金相/无损/焊工信息
- `TEMPLATE_WRITE` / `TEMPLATE_PASS_ROWS`：新模板写入坐标

每条映射的坐标含义见文件内注释。改完后重新运行即可，无需重启。

## 已知限制

- 旧文档结构需与样例一致（7 个表格，行列数固定）。若个别文档结构异常，会标记为「未匹配」并留空，不影响其他文档。
- 复合单元格（如「类别号：Fe-1  组别号：2」）按"标签：值"解析；若旧文档用了非常规分隔符，可能需在 `field_map.py` 调整关键字。
- 电特性实测值（PQR 区）依赖旧文档用 ①②③ 标记多道数据；若旧文档用了其他标记方式，`split_num_marks()` 需相应调整。
